1 Load in time domain

MATLAB usually stores time signals in a matrix variable where columns are signals and rows are their values over time.
For the purposes of the example, three uncorrelated narrowband signals with a length of 200,000 samples and a sampling frequency of 500 Hz were prepared and stored in example_data.mat file.
e_data = load('example_data.mat');
disp(e_data)
s1: [200000×3 double] s2: [200000×3 double] s3: [200000×3 double] s_sampling_freq: 500
Take the 3 columns signal s1, s2 or s3 as the three independent loads varying with time.
Let's see what these signals look like by plotting them out.
L = [100 100 100].*e_data.s3; % load vector L(t)
L_sampling_freq = e_data.s_sampling_freq;
 
plot(1/L_sampling_freq*(1:size(L,1)), L)
xlabel('time, s'), ylabel('load')
legend('L_{1}(t)','L_{2}(t)','L_{3}(t)')
Basic information about the signals: mean, standard deviation and correlation coefficient:

2 Load in frequency domain

Since we have define the load as time signals, the Welch method is used to obtain the PSD matrix of load vector . For this purpose, we use the cpsd(...) function, which allows us to estimate the power spectral density matrix using the Welch method.
[GL, f] = cpsd(L, L, hanning(2^10), 2^9, [], L_sampling_freq, "mimo");
 
% show the PSD matrix
figure
tiledlayout(3, 3)
sp = 0;
for i = 1:3
for j = 1:3
sp = sp+1;
nexttile
plot(f, [real(GL(:,i,j)) imag(GL(:,i,j))])
title(['G_{L' num2str(i) num2str(j) '}(f)'])
end
end
To check the scaling of the PSD matrix, we can compute the area under the PSD functions which are equal to the variances of the particular loads

3 From multiaxial loading to stresses

We calculate the stresses at the point of a structure using the superposition method, which is practical for long time courses.
For the 3-dimensional load process and the six components of the stress tensor , it is necessary to know n × 6 transition coefficients
where
According to the superposition method, the Q matrix is constructed from the stresses obtained on the basis of 3 independent FEM analyzes, each of them for one load of unit value.

Using the matrix of transition coefficients QS and the load vector L, we determine the stress vector S:
S = L*QS; % vector of stress tensor components
plot(S)
xlabel('time, s'), ylabel('stress, MPa')
legend('Sxx','Syy','Szz','Txy','Tyz','Tzx')
Having the stresses at a selected point in the structure (selected node), we can determine any parameters from these stres histories, for example the vector of principal stresses:
p_stress = zeros(size(S,1), 9); %Empty matrix for principal stress vectors coordinates
for i=1:size(S,1)
tens=[S(i,1) S(i,4) S(i,6);...
S(i,4) S(i,2) S(i,5);...
S(i,6) S(i,5) S(i,3)];
[V,D] = eig(tens);
[~,ind] = sort(diag(D));
Ds = D(ind,ind);
Vs = V(:,ind);
P=Vs*Ds;
p_stress(i,1:3) = P(:,1)';
p_stress(i,4:6) = P(:,2)';
p_stress(i,7:9) = P(:,3)';
end
 
% Plotting the principal stress path
% Since these signals are very long we only plot the first 2000 points
ind = 1:2000;
plot3(p_stress(ind,1), p_stress(ind,2), p_stress(ind,3),'r')
hold on
plot3(p_stress(ind,4), p_stress(ind,5), p_stress(ind,6),'g')
plot3(p_stress(ind,7), p_stress(ind,8), p_stress(ind,9),'b')
hold off
axis equal, box on
legend('S1','S2','S3')
From this graph it is easy to see if the stress is:
  • proportional or not,
  • principal stresses change the direction of action or are rather constant.

4 Will we get the same in the frequency domain?

To check this, we will determine the stress PSD matrix directlly in frequency domain using the load PSD and transition coefficients. After that we will compare these values with the stress PSD matrix determined directly from the stress vector.
We will calculate the stress PSD matrix directly from the load PSD matrix:
GS = zeros(size(GL,1),6,6); % empty PSD matrix of stress
for i=1:size(GL,1)
GS(i,:,:) = QS'*squeeze(GL(i,:,:))*QS;
end
GS from from time history:
[GS_from_time, f_from_time] = cpsd(S, S, hanning(2^10), 2^9, [], L_sampling_freq, "mimo");
and plot them out to see if they are the same
subplot(1,2,1)
plot(real(GS(:,1,1)))
title('PSD, frequency domain')
subplot(1,2,2)
plot(real(GS_from_time(:,1,1)))
title('PSD, from time domain')
