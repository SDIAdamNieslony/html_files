1 Problem Statement

The PSD function can be used to generate signal in time domain. This is usually done using the IFFT algorithm with the appropriate input data prepared from the PSD. Following steps are required for this.
  1. Take the sqrt of your PSD. This gives the amplitude spectrum.
  2. Generate uniform noise for the length of the spectrum using rand and multiply by 2*pi. This is a set of random phases.
  3. Make a set of complex numbers using Y=amp.*exp(i*phi);
  4. Replicate complex conj Y=[Y;flipud(conj(Y))];
  5. Give it zero mean Y=[0;Y];
  6. Inverse transform y=ifft(Y);
  7. Scale to get the desired standard deviation.
Of course if your PSD has been ensemble and/or frequency averaged you'll have to interpolate the PSD first to get the length of time series data you desire.
For higer frequency sampling add zeros at the end of the PSD.
% Provide the PSD:
% particular values p and frequency f
p = [0 0 2 2 3 3 0 0 0];
f = [5 8 10 50 60 80 100 110 200];
 
% normalise the PSD for variance = 1
p = p./trapz(f,p);
 
% plot the input PSD
figure
plot(f,p,'-o')
xlabel('frequency f, Hz'), ylabel('PSD')

2 Algorithm

The algorithm is divided into several steps. We need the initial information, i.e. the sampling frequency of the signal and the number of samples to generate.
Of course, the main input value is PSD but you need to check the correctness of its definition. For example it must have a value at f=0 Hz for interpolation. We check it and, if necessary, add this value by copying the first value from p. In addition, it must be defined:
  1. Sample rate fs in Hz for the signal output,
  2. Signal length sl in samples,
  3. Correlation coefficient cc which determines the correlation between two signals.
fs =940; % Hz
sl =25810; % samples
cc =-0.48; % dimensionless

Step 1. Input data validation and PSD preparation

The input data must be verified to be complete in order to generate a signal using the numerical method, among other things, it must be checked whether the value given is at frequency 0 and at the maximum frequency resulting from the sampling frequency.
p = p(:); % vectorization
f = f(:); % vectorization
 
% add PSD value at the frequency f = 0 Hz
if f(1)~=0
f = [0; f];
p = [p(1); p];
end
 
% PSD length
psdl = fix(sl/2);
% interpolation points (Hz) for PSD
f1 = linspace(0, psdl*fs/sl, psdl)';
 
% add PSD = 0 at the highest frequency of f1
if f1(end) > f(end)
p(end+1) = p(end);
f(end+1) = f1(end);
end
 
% PSD points for each frequency in f1 using interpolation
p1 = interp1(f, p, f1, 'linear');
 
% logarithmic interpolation, your choice
% p1 = -1 + exp(interp1(f, log(1 + p), f1, 'linear'));
 
figure
tiledlayout(2,1)
 
nexttile
plot(f,p,'bo-'), title('PSD before interpolation')
 
nexttile
plot(f1,p1,'r.'), title('PSD after interpolation')

Step 2. Signal generation using IFFT

Properly prepared Power Spectral Density Function will be used to generate signals using the Inverse Fourier Transform method. The input to the IFFT is a vector of complex numbers that reflect the amplitudes and phase shifts of the harmonic components.
% amplitudes of harmonic components
amp1 = sqrt(2*p1.*f1(2));
 
% phases of harmonic components
phi1 = 2*pi*rand(length(amp1), 1); % fazy dla amplitud
 
% input as a complex numbers
Y1 = amp1.*exp(1i*phi1);
 
% mirror reflection for negative frequencies
Y1 = [0; Y1; flipud(conj(Y1))];
 
% signal generation using the inverse Fourier transform
sig1 = ifft(Y1);
To generate a similar signal a second time with the same PSD and sampling parameters, we can use the prepared data and just generate the phase shifts again.
% same amplitudes but new phase shifts
phi2 = 2*pi*rand(length(amp1), 1);
 
Y2 = amp1.*exp(1i*phi2);
Y2 = [0; Y2; flipud(conj(Y2))];
 
% secound signal
sig2 = ifft(Y2);
Let's check it out the correlation between sig1 and sig2:
disp(corrcoef([sig1 sig2]))
1.0000 0.0020 0.0020 1.0000
Of course, both signals are not correlated because the generation of phase shifts was performed independently by a random number generator with a uniform distribution.

Step 3. Obtaining signals with a given correlation

Method 1: generation of a signal with a phase shift of the harmonic components

We use the definition of the correlation coefficient whose inverse cosine in radians arccos(x) value is equal to the phase shift of two harmonic signals.
Eq. (1)
% new phases of harmonic components
phi3 = acos(cc) + phi1;
 
% generation of a third signal with appropriate correlation to the first one
Y3 = amp1.*exp(1i*phi3);
Y3 = [0; Y3; flipud(conj(Y3))];
sig3 = ifft(Y3);
 
% checking the correlation coefficient
disp(corrcoef([sig1 sig3]))
1.0000 -0.4800 -0.4800 1.0000
In this way we get two signals correlated with each other by a given value of the correlation coefficient, we can see these signals on the figure below
figure
plot([sig1 sig3])

Method 2: summation of two non-correlated signals

Two random processes and correlated respectively to each other at the level of the correlation coefficient can be obtained by appropriately summing the two uncorrelated processes and as follows
The correlation coefficient is determined as follows (J. S. Bendat i A. G. Piersol, Random Data: Analysis and Measurement Procedures. John Wiley and Sons, 2010)
Since the variance (coefficients w1 and w2 to preserve the variance), the standard deviation is = 1. The standard deviation of the second process is w1 * 1
several lines of mathematical derivation ...
Implementation in MATLAB
% coefficient a as a function of correlation
a = (cc*(abs(cc)-sqrt(1-cc^2)))/(2*cc^2-1);
 
w1 = a./sqrt(a.^2+(1-abs(a)).^2);
w2 = (1-abs(a))./sqrt(a.^2+(1-abs(a)).^2);
 
% signal sig4 as a combination of two uncorrelated signals sig1 and sig2
sig4 = w1*sig1 + w2*sig2;
 
disp(corrcoef([sig1 sig4]))
1.0000 -0.4787 -0.4787 1.0000
%
figure, plot([sig1 sig4])

Method 3: using Cholesky factorization

The most efficient method of generating signals with a given correlation uses the Cholski decomposition to determine the weights used to add the uncorrelated signals. To use this method, you need to define a correlation matrix first for 3 signals
c = [1 cc -.3; cc 1 .7; -.3 .7 1]; % correlation matrix
disp(c)
1.0000 -0.4800 -0.3000 -0.4800 1.0000 0.7000 -0.3000 0.7000 1.0000
For 3 correlated signals we need 3 uncorrelated signals as the basis
% same amplitudes but new phase shifts
phi5 = 2*pi*rand(length(amp1), 1);
 
Y5 = amp1.*exp(1i*phi5);
Y5 = [0; Y5; flipud(conj(Y5))];
sig5 = ifft(Y5);
 
% signals in one variable (3 columns)
sig = [sig1 sig2 sig5];
disp(corrcoef(sig))
1.0000 0.0020 0.0175 0.0020 1.0000 0.0099 0.0175 0.0099 1.0000
 
% and final matrix operation
sigcorr = sig*cholcov(c);
 
% the result
disp(corrcoef(sigcorr))
1.0000 -0.4787 -0.2862 -0.4787 1.0000 0.6994 -0.2862 0.6994 1.0000
plot(sigcorr)
