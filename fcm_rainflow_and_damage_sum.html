1 Rainflow counting

The rainflow counting method introduced in 1968 by Matsuishi and Endo is the first accepted method used to isolate closed load reversals or cycles for fatigue analysis. The name "rainflow" comes from comparing this method to the flow of rain falling on a pagoda and running off the edges of the roof. The algorithm is applied to a single-channel signal (uniaxial load) and there are many variations of this algorithm, however, the differences between them are minor and will not be discussed in detail here. The rainflow method is directly related to fatigue damage summation theories because this algorithm allows the extracting of load cycles that we later use to sum the fatigue damage.

1.1 Simple signal with a few cycles

The easiest way to effectively use the rainflow algorithm in MATLAB is to use the rainflow function included in Signal Processing Toolbox. Please see Matlab help for detailed information about this function.
The rainflow script can be run in several ways, but the simplest is to use just one variable representing the time history to analyze. The output variable will be a matrix of n rows for each designated cycle or half-cycle. The 5 columns contain:
  • counts, 1.0 for cycle and 0.5 for half cycle,
  • ranges, for amplitude divide by 2,
  • mean values,
  • initial sample indices,
  • and final sample indices.
See below for an example of a simple signal. Notice that only the reversals are involved in the cycle counting.
% simple signal 10 samples long
x = fcm_gen_load(1);
rf = rainflow(x);
array2table(rf,'VariableNames',{'Count','Range','Mean','Start','End'})

We can show graphical cycles by connecting with dashed lines the points where the rainflow algorithm has determined cycles and half-cycles. Different colors are used for successive cycles to distinguish them.
figure
 
col = 'ymcrgb'; % variable the colors
plot(x, 'k') % plot the counted time history in black
hold on
% for each counted cycle
for c = 1:length(rf)
colnr = rem(c-1,6) + 1;
xc = rf(c, 4:5);
yc = x(rf(c, 4:5))';
if rf(c, 1) == 0.5
% 0.5 cycle, UP and DOWN
plot(xc, yc, 'x--','Color', col(colnr))
elseif yc(1)>yc(2)
% 1 cycle DOWN
ind = rf(c,5);
while x(ind)<x(rf(c,4))
ind = ind+1;
end
plot([xc ind], [yc yc(1)], 'o--','Color',col(colnr))
else
% 1 cycle UP
ind = rf(c,5);
while x(ind)>x(rf(c,4))
ind = ind+1;
end
plot([xc ind], [yc yc(1)], 's--','Color',col(colnr))
end
end
hold off
xlabel('samples')
ylabel('signal, cycles and half-cycles')

1.2 Long signals

In the case of long load runs, in which the rainflow algorithm finds a significant number of cycles, the result of such counting is presented using the so-called rainflow matrix. If numerical values of the rainflow matrix are needed for calculations, the histcounts2 function can be used. Since rainflow finds a cycle (1.0) or half-cycle (0.5) we can't use histogram2 directly. So we'll use histcounts2 twice and plot with bar3 taking into account also half-cycles.
% signal 10000 samples long, see fcm_gen_load for details
x = fcm_gen_load(2);
rf = rainflow(x);
 
figure
cycleCount = rf(:,1);
cycleAmp = 0.5*rf(:,2);
cycleMean = rf(:,3);
 
% count all cycles and get the bounds
[N1, Xedges, Yedges] = histcounts2(cycleAmp, cycleMean);
 
% count only half-cycles
ind = cycleCount==0.5;
N2 = histcounts2(cycleAmp(ind), cycleMean(ind), Xedges, Yedges);
 
% subtract half cycles from full histogram
N = N1 - 0.5*N2;
bar3(N, 1.0);
xlabel('cycle ampitude (class number)')
ylabel('cycle mean (class number)')
The rainflow function also allows us to graphically represent the rainflow matrix, it should be called without assigning output variables.
figure
rainflow(x)
Rainflow function also allows to calculate rainflow matrix as numeric values. It should be called with appropriate output variables.
[c,rm,rmr,rmm,idx] = rainflow(x);
figure
bar3_for_histcounts2(0.5*rmr, rmm, rm)
xlabel('cycle amplitude')
ylabel('cycle mean')

2 Damage summation

Miner's rule, also known as the linear damage rule, is used to calculate the cumulative damage that occurs under a load of variable amplitude. It provides a usefull approach to predicting the crack initiation by summing the damage caused by individual stress or strain cycles. Miner's rule is based on the assumption that each loading cycle (stress or strain) contributes a fraction of the total damage required to cause failure. 2.1 Example: two level loading

Let's assume we have a load consisting of two blocks with different amplitudes and a given number of cycles for each block. Knowing the number of cycles and amplitudes, we can calculate the damages according to Meiner's rule.
% two bloks of loading
b1Amp = 400; % MPa
b1NCycle = 6000; % cycles
b2Amp = 300; % MPa
b2NCycle = 100000; % cycles
 
% SN curve
Saf = 250; % MPa, fatigue limit
Nf = 2e6; % number of cycle for fatigue limit
m = 8; % slope of the SN curve
B = Nf/(Saf^-m); % SN curve constant
Suts = 900; %
 
% inline function to get the number of cycles from the SN curve
NfromSnCurve = @(Sai) B*Sai.^(-m);
 
% damage
D1 = b1NCycle/NfromSnCurve(b1Amp);
D2 = b2NCycle/NfromSnCurve(b2Amp);
D = D1 + D2;
 
% display the results
disp(table(D, D1, D2))
D D1 D2 _______ _______ _______ 0.34384 0.12885 0.21499
We see values for partial damage and the sum. Graphical interpretation below.

2.2 Example: random loading using rainflow

In practice, fatigue damage summation is most often used for random or variable amplitude loading after counting cycles using the rainflow method. In this case, we are dealing with a significant number of cycles, and it is recomended to perform the fatigue damage summation for each designated cycle. This procedure also allows for taking into account the mean value for a given cycle. Example below.
% load generation
x = fcm_gen_load(3);
% rainflow
rf = rainflow(x);
 
% based on previously defined SN curve:
NCycle = rf(:,1); % number of cycles
Sa = 0.5*rf(:,2); % amplitudes
Sm = rf(:,3); % mean stress
% mean stress effect correction according Goodman
KGoodman = 1/(1-(Sm/Suts));
AmpTrans = Sa*KGoodman;
% damage summation
D = sum(NCycle./NfromSnCurve(Sa));
disp(table(D))
D ________ 0.030942
We can draw the rainflow matrix by following the procedure shown earlier. Hoever, more interesting from the point of view of material fatigue is the so called damage matrix, which indicates the most damaging cycles. From this matrix we can find out at what load levels the random run causes more damage.
[rf,rm,rmr,rmm] = rainflow(x);
 
figure
t = tiledlayout(1,2);
 
nexttile(t)
bar3_for_histcounts2(0.5*rmr, rmm, rm)
xlabel('cycle amplitude')
ylabel('cycle mean')
 
nexttile(t)
% initialising the dmamage matrix
rmDamage = zeros(size(rm));
for r = 1:(length(rmr)-1)
for m = 1:(length(rmm)-1)
Sa = 0.5*(rmr(r)+rmr(r+1))/2;
Sm = (rmm(m)+rmm(m+1))/2;
% mean stress effect correction according Goodman
KGoodman = 1/(1-(Sm/Suts));
AmpTrans = Sa*KGoodman;
rmDamage(r,m) = rm(r,m)./NfromSnCurve(AmpTrans);
end
end
 
bar3_for_histcounts2(0.5*rmr, rmm, rmDamage)
xlabel('cycle amplitude')
ylabel('cycle mean')
To control the number of classes in the rainflow matrix, use the histcounts2 function. Below is an example of how to properly prepare a rainflow matrix with given classes for ampltiude and mean stress.
cycleCount = rf(:,1);
cycleAmp = 0.5*rf(:,2);
cycleMean = rf(:,3);
[N1,Xedges,Yedges] = histcounts2(cycleAmp, cycleMean, [10 12]);
 
% count only half-cycles
ind = cycleCount==0.5;
N2 = histcounts2(cycleAmp(ind), cycleMean(ind), Xedges, Yedges);
 
% subtract half cycles from full histogram
N = N1 - 0.5*N2;
 
figure
t = tiledlayout(1,2);
 
nexttile(t)
bar3_for_histcounts2(Xedges, Yedges, N)
xlabel('cycle ampitude (class number)')
ylabel('cycle mean (class number)')
 
nexttile(t)
% initialising the dmamage matrix
NDamage = 0*N;
for r = 1:(length(Xedges)-1)
for m = 1:(length(Yedges)-1)
Sa = (Xedges(r)+Xedges(r+1))/2;
Sm = (Yedges(m)+Yedges(m+1))/2;
% mean stress effect correction according Goodman
KGoodman = 1/(1-(Sm/Suts));
AmpTrans = Sa*KGoodman;
NDamage(r,m) = N(r,m)./NfromSnCurve(AmpTrans);
end
end
bar3_for_histcounts2(Xedges, Yedges, NDamage)
xlabel('cycle amplitude')
ylabel('cycle mean')
